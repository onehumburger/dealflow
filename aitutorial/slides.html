两年发生了什么

  • 2024年初：大多数人刚开始用ChatGPT聊天
  • 2026年初：AI可以自主完成复杂任务、调用工具、读取整本合同
  • 变化速度比你想象的快
用2024到2026的时间线，让律师感受到变化的加速度

上下文窗口的飞跃

  • Token（词元）= AI读取文本的最小单位
  • 中文大约 1个字 ≈ 1-2个token
  • 4K tokens ≈ 3页A4纸 → 200K tokens ≈ 一整本书（500页）
  • 实际意义：整份合同、整套案卷可以一次性丢给AI分析
Token是AI处理文本的基本单位，可以理解为AI的"字"。中文一个字大约等于1-2个token。 以前AI只能看3页纸，现在能看一整本500页的书——这意味着整份合同、整套案卷不用拆分，直接丢给AI就行。

多模态：不只是文字

  • 读图片：拍一张合同照片发给AI，它能识别并分析内容
  • 读PDF：直接上传排版复杂的判决书，AI能理解其中的结构和逻辑
  • 读表格：上传Excel财务报表，AI能计算、对比、找异常
  • 实际场景：收到对方扫描版合同，不用手动OCR，直接丢给AI
以前必须把文字一段段粘贴给AI，现在直接丢文件。 重点强调：扫描件不用再手动OCR了，AI直接能读图片里的文字。

推理能力的突破

  • 不是"背答案" → 而是能"想问题"
  • 举例：给AI一份合同的违约金条款，它不只会告诉你"有违约金条款"
  • 它会分析：违约金比例是否过高、是否可能被法院调整、相比行业惯例是否合理
  • 还能发现矛盾：比如第3条和第7条的约定互相冲突
这是质变——从信息检索到逻辑推理。 用违约金条款的例子让律师直观感受：AI不只是搜索工具，它能做初步的法律分析。

你现在用的AI是这样的

  • 你问一句，它答一句
  • 像微信聊天
  • 这叫 Chat（对话模式）
大多数律师停留在这个阶段，包括用豆包

但AI可以是这样的

  • 你给一个目标："帮我审查这份合同的所有风险点"
  • Agent 自己会：先通读全文 → 识别关键条款 → 逐条分析风险 → 生成结构化报告
  • 中间不需要你做任何操作
  • 这叫 Agent（智能代理）— AI从"回答问题"升级为"完成任务"
Agent是今天最重要的概念——从工具到助手的转变。 关键区别：Chat是你问一句它答一句，Agent是你给目标它自己完成整个任务。

Plugin（插件）

  • 裸机AI = 通用能力，AI + 工具 = 专业能力。三个关键概念：
  • 给AI接上专业数据源和能力
  • 举例：Claude 装上法律插件后，可以直接检索法规数据库
  • 举例：装上翻译插件，可以一边审查英文合同一边输出中文分析
  • 类比：就像手机装了高德地图，手机本身不认路，装上APP就会导航
Plugin是最好理解的——就像手机装APP。 手机本身不会导航，但装了高德地图就能导航。AI本身不连数据库，装了插件就能检索法规。

Skill（预设技能模板）

  • = 专家级提示词 + 预设工作流，打包在一起
  • 没有Skill：你要自己写一大段指令告诉AI怎么审查合同
  • 有了Skill：一键启动"合同审查"模式，AI自动按照预设步骤执行
  • 类比：Skill就像律所的标准作业流程（SOP），但是给AI用的
Skill解决了"不会写prompt"的问题。 就像律所有SOP，新人按流程做就不容易出错。Skill就是AI的SOP，让AI按预设的专业流程来工作。

MCP（Model Context Protocol）

  • 模型上下文协议 — 让AI直接连接你电脑上的文件和系统
  • 举例：AI直接读取你桌面上的合同文件夹，批量分析100份合同
  • 举例：连接律所的案件管理系统，AI可以查阅历史案件
  • 现状：技术刚成熟，潜力巨大，是未来AI工作的基础设施
MCP比较技术，点到为止，让他们知道存在就行。 核心信息：MCP让AI不再局限于网页聊天框，而是能直接操作你电脑上的文件和系统。

AI 助力律师工作

方法论与实践建议

核心章节，从方法论角度讲AI如何融入律师日常工作

哪些工作适合交给AI？

适合 AI

  • 重复性高的工作
  • 有明确标准和流程的
  • 信息量大需要梳理的
  • 初稿 / 草稿类工作

不适合 AI

  • 需要人情世故的沟通
  • 需要创造性法律策略的
  • 涉及敏感谈判策略的
  • 最终拍板决策

AI做80%的体力活，律师做20%的脑力活

帮律师建立判断框架：不是所有工作都丢给AI，而是识别哪些环节可以AI化。 80/20法则：AI处理大量重复性的初步工作，律师集中精力做判断和决策。

批量处理思维

从"一次处理一份"到"一次处理一批"

传统方式

100份供应商合同
→ 一份份打开、逐份检查
→ 2天

AI方式

100份供应商合同
→ 全部上传，一句指令
→ 30分钟

指令示例："检查每份合同的争议解决条款，列出哪些缺少仲裁条款"

这是思维方式的转变。很多律师还是一份一份处理文件，没想过可以批量操作。 尽调中批量提取公司关键信息（注册资本、股东结构、诉讼记录）也是同理。

AI辅助多语言工作

跨境交易场景：审查 + 翻译 + 分析，一步完成

场景：收到一份英文 SPA（股权购买协议）

以前：先翻译成中文 → 再审查分析 → 两步走，耗时长

现在：一个指令，AI同时完成 →

  • 用英文标注合同风险条款
  • 用中文输出给国内客户的风险提示备忘录
跨境业务的律师会特别有感触。以前翻译和分析是两个步骤，现在合并为一步。 而且AI的翻译质量在法律语境下已经相当好了。

AI辅助新人培养

Junior律师的AI导师

  • 做完审查，让AI复查：做完一份合同审查 → 让AI指出遗漏的风险点
  • 写完文书，让AI校对：写完法律意见书 → 让AI检查逻辑漏洞和表述问题
  • 遇到新领域，让AI梳理：不熟悉的法律领域 → 让AI做背景知识整理

核心价值：加速新人成长，不占用 Senior 律师的时间

这个话题合伙人会特别感兴趣——降低带新人的时间成本。 AI不能替代senior的指导，但可以承担大量"基础问题解答"和"初步审查反馈"的工作。

提示词决定输出质量

  1. 角色设定："你是一名中国执业律师，专注于劳动法…"
  2. 结构化指令：明确输出格式（分条列出、先结论后分析）
  3. 分步拆解：复杂任务拆成多步，逐步推进

模糊指令："帮我看看这个合同" → 精确指令："你是一名资深商事律师，请审查以下合同，重点关注违约金、竞业禁止和管辖权条款，逐条列出风险并给出修改建议"

使用AI的注意事项

  • 数据隐私：客户数据上传需考虑合规（数据出境、保密义务）
  • AI幻觉：AI会一本正经地编造不存在的判例
    真实案例：有美国律师引用了AI编造的6个判例，被法院制裁
  • 核心原则：AI输出 = 初稿，不是终稿。每一条引用都必须人工校验

讨论：你们觉得客户数据上传AI有什么合规考量？

轻量处理，让律师自己讨论。 AI幻觉的例子：2023年美国律师Steven Schwartz用ChatGPT生成的诉状中引用了6个完全不存在的判例，被法院罚款5000美元。 这个案例说明：AI的输出必须校验，尤其是判例引用。
