Part 1

认知升级

两年发生了什么

  • 2024年初：大多数人刚开始用ChatGPT聊天
  • 2026年初：AI可以自主完成复杂任务、调用工具、读取整本合同
  • 变化速度比你想象的快
用2024到2026的时间线，让律师感受到变化的加速度

上下文窗口的飞跃

  • 4K tokens → 200K tokens
  • 4K ≈ 3页A4纸 / 200K ≈ 一整本书（约500页）
  • 意味着：整份合同、整套案卷可以一次性丢给AI
这是最直观的进步——以前AI只能看几页，现在能看整本书

多模态：不只是文字

  • 读图片：合同扫描件、证据照片
  • 读PDF：直接解析排版复杂的法律文书
  • 读表格：财务报表、股权结构图
以前必须把文字一段段粘贴给AI，现在直接丢文件

推理能力的突破

  • 不是"背答案" → 而是能"想问题"
  • 能分析利弊、发现矛盾、提出反驳
  • 对律师意味着：AI可以做初步的法律分析，不只是搜索
这是质变——从信息检索到逻辑推理

你现在用的AI是这样的

  • 你问一句，它答一句
  • 像微信聊天
  • 这叫 Chat（对话模式）
大多数律师停留在这个阶段，包括用豆包

但AI可以是这样的

  • 你给一个目标，它自己规划步骤
  • 自动调用搜索、读取文件、生成文档
  • 这叫 Agent（智能代理）
Agent是今天最重要的概念——从工具到助手的转变

一个类比

Chat = 实习生

你说一步，他做一步

Agent = 资深助理

你说目标，他安排整个流程

这个类比是今天最重要的takeaway之一

同一个任务，两种方式

任务：审查一份合同中的竞业禁止条款

  • Chat 方式：逐段粘贴，手动追问
  • Agent 方式：上传合同，一句话指令，自动完成
Demo步骤： 1. 先用豆包/ChatGPT网页版，手动粘贴合同段落，问"这个条款有什么风险" 2. 再用Claude Agent模式，上传整份合同，说"审查这份合同的竞业禁止条款，标注风险点" 3. 对比两者的效率和输出质量

手机 vs 手机 + APP

  • 裸机AI = 通用能力
  • AI + 工具 = 专业能力
  • 三个关键概念：Plugin / Skill / MCP
用手机APP类比，律师马上就懂了

Plugin（插件）

  • 给AI接上专业数据源
  • 例：法律数据库插件 → AI能搜判例
  • 例：财务插件 → AI能读财报
  • 你不需要自己做，装上就行
Plugin是最好理解的——就像手机装APP

Skill（预设技能模板）

  • = 专家级提示词 + 预设工作流
  • 例：一个"合同审查 Skill" 预设了审查步骤、关注要点、输出格式
  • 你不需要成为 prompt 专家
Skill解决了"不会写prompt"的问题

MCP（Model Context Protocol）

  • 让AI连接你的本地文件和系统
  • 例：连接律所的案件管理系统
  • 例：读取本地文件夹里的所有合同
  • 技术在快速成熟，未来潜力巨大
MCP比较技术，点到为止，让他们知道存在就行

工具太多，怎么选？

  • 先分清类别：通用 vs 法律专用
  • 不需要全部会，选1-2个深度用
帮律师降低选择焦虑

通用AI工具对比

豆包：中文好，免费，大家已经在用
Kimi：长文档能力强，适合读合同
Claude：推理强、上下文长、有法律 Plugin
ChatGPT：生态最大，Plugin 最多
不需要记住所有，重点推荐Kimi和Claude

法律专用工具

  • OpenClaw：开源法律AI
  • Harvey AI / CoCounsel：国际大所在用
  • Claude Legal Plugin：Claude的法律专业扩展
法律专用工具是趋势，虽然国内生态还在早期

NotebookLM

  • Google出品，免费
  • 上传文档 → 自动生成摘要、问答、播客式讲解
  • 律师场景：100页案卷 → 案情摘要 → 播客给客户听
  • 门槛极低，推荐作为入门第一个AI工具
这是今天最推荐的入门工具，现场演示
